## 学习内容

### Prompt（提示词）— 今日核心 🔑

> **第一性原理**：Prompt 是软约束，不是安全边界。真正的边界必须由代码、权限、校验和审计来承担。

**为什么重要？**
- 作为「内容运营」方向，你需要频繁和 AI 交互来产出内容——学会写好 Prompt，就是学会了「指挥 AI 的接口设计」
- Prompt 不是越长越好，而是要把任务目标、输入边界、输出格式、失败处理写清楚

**核心知识节点：**
1. **Instruction**（指令）：告诉模型角色、任务、禁止行为、输出格式
   - 四段式写法：任务目标 → 可用输入 → 禁止行为 → 输出格式和失败格式
2. **Few-shot**（少样本示例）：放少量示例，让模型模仿判断和格式
3. **Structured Output**（结构化输出）：用 JSON schema 承载结果，让后续代码可检查
4. **Prompt Injection**（提示注入）：攻击者通过输入让模型忽略规则——在 Agent 场景尤其危险

**今日练习建议：**
- 为你的「每日学习笔记」写一个固定 Prompt 模板，让 Agent 每次按统一格式输出
- 试试给 Hermes 一段包含"结构化输出要求"的 Prompt，观察它返回的格式差异

### Context（上下文）— 推荐延伸

> **第一性原理**：模型只能基于它看见的上下文行动；系统必须决定什么能进上下文、带着什么身份进去、过期后怎么退出。

**核心知识节点：**
1. **Context Window**：模型一次能看到的范围——不是塞越多越好，关键信息要前置
2. **Context Engineering**：设计上下文的信息层级——指令层、任务层、事实层、知识层、记忆层
3. **Memory**：跨请求保留的信息（偏好、历史）——但不能替代实时授权
4. **Knowledge Base**：外部知识库——需要维护来源、时间、版本

**今日练习建议：**
- 观察你今天和 Hermes 的对话：哪些信息被放入了上下文？它们属于哪个层级？
- 回顾前两天的学习：Agent 是如何记住你的学员画像和学习计划的？

### RAG（检索增强生成）— 挑战延伸

- RAG 让模型在回答前先从外部知识库检索相关资料
- 适合解决「模型知识过期」的问题
- 对内容运营方向特别有用：可以喂给模型参考资料，让它基于准确素材生成内容
